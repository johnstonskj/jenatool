Managing RDF Content

The tool recognizes a number of specific file formats and content types, as shown in the following table. These contents types fall into two categories, RDF Content and SPARQL Queries. Each of these categories has associated actions that users can perform on the resources in the workspace as described later.

Content Types

The following table summarizes the supported content types.

Content TypeFile Extension(s)Eclipse EditorComments
SPARQL Query.sparqltextA SPARQL query.
N-Triples RDF Content.nttextRDF content in N-Triple format.
Turtle RDF Content.ttltextRDF content in Turtle format.
N3 RDF Content.n3textRDF content in N3 format.
RDF/XML RDF Content.rdfxmlRDF content in RDF/XML format.

The RDF Content types support import actions so that the contained RDF data can be added to a Jena store; the Query content type(s) support a run action to execute the query against a Jena store.

New Content Wizards

At this time only the RDF/XML content type is supported by the standard Eclipse File->New menu. When you select File->New->Other... you will see the following dialog, notice the open Jena category.

When you select the New RDF/XML File option you'll be presented with the specific dialog shown below.

The first two inputs are pretty standard Eclipse fair, the last is specific to RDF and is used in creating the default content, as shown below.

Container
the workspace container in which the resource will be created.
File name
the name of the workspace resource to create.
Subject URI
the URI to be the subject of the graph, can be left blank.

When you hit Finish the tool will create a new resource in your workspace and expand a simple template into the resource to get you started. Note that the attrbute rdf:about will be set to the value you specified for Subject URI in the dialog above, in this case it is the empty string. 

<?xml version="1.0"?>
<rdf:RDF xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
         xmlns:dc="http://purl.org/dc/elements/1.1/">
  <rdf:Description rdf:about="">
  </rdf:Description>
</rdf:RDF>
